NYC Permitted Event Information (tvpp-9vvx)
- Official NYC Open Data dataset for approved event applications expected within the next month.
- Dataset page says permitted film events only reflect those affecting one or more streets for at least five days.
- Data provided by Office of Citywide Event Coordination and Management (CECM).
- Updated daily; page shows last updated Apr 16 2026.
- Relevant fields confirmed on dataset page: event_id, event_name, start_date_time, end_date_time, event_agency, event_type, event_borough, event_location, event_street_side, street_closure_type, community_board, police_precinct, cemsid.

Street Resurfacing Schedule (xnfm-u3k5)
- Official NYC Open Data dataset for streets where crews will be doing milling or resurfacing work each week.
- Data provided by Department of Transportation (DOT).
- Dataset page shows daily updates, with note that data may be sporadic during winter/off-season.
- Relevant fields confirmed on dataset page: workscheduleprojectlocationid, boroughname, day, date, onstreetname, fromstreetname, tostreetname, communityboard, area, worktype, crewtype, shifttype, oftcode, locationsegmentid, locationwkt, locationnodeid.
- Dataset page notes schedules are subject to change due to weather or equipment issues.

Additional mapping findings:
The Parks Permit Areas dataset is relevant to permitted events because it includes both CEMSID and permit-area geometry metadata for parks-managed event spaces. That creates a stable way to map most CB6 event rows from the permitted-events feed, since the attached CB6 event file is overwhelmingly Parks Department activity and includes CEMSID values for those locations. For non-park events such as Street Activity Permit Office or Police Department rows, the event location text can be handled separately with street-based geocoding or simpler text-based mapping fallbacks.

The permitted-events dataset should be treated as active across each row's start and end datetime window. The resurfacing schedule should be treated as active on the listed schedule date for each row, since the official dataset is organized as dated work occurrences rather than a multi-day validity window. The events dataset updates daily and reflects applications occurring within the next month, while the resurfacing dataset updates daily and represents weekly milling or paving schedules subject to change.
